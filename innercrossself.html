SQL JOIN Növləri: CROSS JOIN və SELF JOIN

İlişkili verilənlər bazası sorğuları üçün dərin baxış

Giriş: SQL JOIN Növləri

SQL-də JOIN əməliyyatları müxtəlif cədvəllərdən məlumatları birləşdirmək üçün istifadə olunur. Bu təqdimatda iki xüsusi JOIN növünü - CROSS JOIN və SELF JOIN-i ətraflı şəkildə araşdıracağıq.

📌 Əhəmiyyətli: JOIN əməliyyatları verilənlər bazasında ən çox istifadə olunan əməliyyatlardandır və onları düzgün başa düşmək performanslı sorğular yazmaq üçün çox vacibdir.

Təqdimat Məzmunu:

  • CROSS JOIN nədir və necə işləyir?
  • SELF JOIN nədir və hansı hallarda istifadə olunur?
  • Hər iki JOIN növünün sintaksisləri
  • Praktik nümunələr və kod parçaları
  • İki JOIN növü arasındakı fərqlər
  • Praktik tətbiqlər və məsləhətlər

1. CROSS JOIN Nədir?

CROSS JOIN iki cədvəlin bütün mümkün kombinasiyalarını yaradan JOIN növüdür.

Xüsusiyyətləri:

  • Dekart hasilini yaradır
  • Ümumiyyətlə şərt tələb etmir (WHERE olmadan)
  • Nəticədə M×N sayda sətir əldə edilir:
    • M: birinci cədvəlin sətirlərinin sayı
    • N: ikinci cədvəlin sətirlərinin sayı
  • Böyük cədvəllər üzərində performansa ciddi təsir göstərə bilər
⚠️ Diqqət: CROSS JOIN böyük cədvəllər üzərində istifadə edilərsə, nəticə çox böyük ola bilər. Məsələn, 1000 sətirli bir cədvəli 1000 sətirli başqa bir cədvəllə CROSS JOIN etsək, 1.000.000 sətir nəticə alarıq!

2. CROSS JOIN Sintaksisi

CROSS JOIN-in iki əsas sintaksis forması var:

Açıq sintaksis (explicit):

SELECT sütun_adları
FROM cədvəl1
CROSS JOIN cədvəl2;

Qapalı sintaksis (implicit):

SELECT sütun_adları
FROM cədvəl1, cədvəl2;
💡 Professional məsləhət: Açıq sintaksisdən (CROSS JOIN açar sözləri ilə) istifadə etmək daha yaxşıdır, çünki kodun oxunaqlılığını artırır və başqa developerlər üçün aydın olur ki, burada nə edilir.

4. SELF JOIN Nədir?

SELF JOIN bir cədvəlin özü ilə JOIN edilməsidir. Bu, eyni cədvəli iki fərqli aliases (ləqəb) adı ilə istifadə edərək həyata keçirilir.

Əsas tətbiq halları:

  • İyerarxik məlumat strukturları (menecer-işçi münasibətləri)
  • Tərkibində özünə istinad olan məlumatlar
  • Ardıcıl qeydlərin müqayisəsi (məsələn, tarixlər üzrə)
  • Qraf traversalları (dost şəbəkələri, təşkilat quruluşları)
📌 Qeyd: SELF JOIN texniki olaraq ayrı bir JOIN növü deyil, daha çox bir cədvəlin özü ilə INNER JOIN, LEFT JOIN və ya digər JOIN növlərinin istifadə olunmasıdır.

5. SELF JOIN Sintaksisi

SELF JOIN üçün cədvəli iki fərqli alias (ləqəb) ilə istifadə etmək lazımdır:

SELECT A.sütun_adları, B.sütun_adları
FROM cədvəl_adı AS A
INNER JOIN cədvəl_adı AS B
ON A.açar_sütun = B.açar_sütun
WHERE [əlavə şərtlər];

Digər JOIN növləri ilə SELF JOIN:

-- LEFT JOIN ilə SELF JOIN
SELECT A.ad, B.ad
FROM İşçilər A
LEFT JOIN İşçilər B
ON A.menecer_id = B.id;
-- INNER JOIN ilə SELF JOIN
SELECT A.ad, B.ad
FROM İşçilər A
INNER JOIN İşçilər B
ON A.departament_id = B.departament_id
WHERE A.id != B.id;
⚠️ Diqqət: SELF JOIN-də alias istifadə etmək məcburidir, əks halda SQL server sütunların hansı cədvələ aid olduğunu anlaya bilməz.

7. CROSS JOIN vs SELF JOIN

İki JOIN növü arasındakı əsas fərqlər:

Xüsusiyyət CROSS JOIN SELF JOIN
Cədvəl sayı 2 fərqli cədvəl 1 cədvəl (özü ilə)
Nəticə sayı M×N M×M (potensial olaraq)
Şərt Adətən şərtsiz Həmişə şərtlə (ON)
Əsas tətbiq Kombinasiya yaratmaq İçəri münasibətlər
Performans təsiri Yüksək (böyük cədvəllərdə) Orta
Alias (ləqəb) zəruriliyi Tələb deyil Məcburidir

Hansını nə vaxt istifadə etməli?

  • CROSS JOIN → Bütün mümkün kombinasiyalara ehtiyacınız olduqda
  • SELF JOIN → Bir cədvəlin daxilindəki əlaqələri araşdırmaq lazım olduqda

8. Praktik Tətbiqlər

CROSS JOIN tətbiqləri:

  • ✅ Bütün mümkün variantların generasiyası - Məsələn, bütün rəng və ölçü kombinasiyaları
  • ✅ Test məlumatlarının yaradılması - Müxtəlif parametrlər üçün test dataları
  • ✅ Kombinator məsələləri - Ehtimal hesablamaları, statistik analiz
  • ✅ Tarix aralıqlarının generasiyası - Bütün gün və saat kombinasiyaları

SELF JOIN tətbiqləri:

  • ✅ İyerarxik strukturlar - Menecer-işçi münasibətləri, təşkilat quruluşu
  • ✅ Ardıcıl tarixlərin müqayisəsi - Əvvəlki/gələn günlərin məlumatlarını müqayisə
  • ✅ Qraf traversalları - Dost şəbəkələri, sosial media əlaqələri
  • ✅ Eyni kateqoriyadakı məhsullar - Oxşar məhsulların qruplaşdırılması
  • ✅ İşçilərin cütləşdirilməsi - Eyni ixtisaslı işçilərin tapılması

9. Əhəmiyyətli Qeydlər

CROSS JOIN üçün:

  • 📏 Böyük cədvəllərdə diqqətli istifadə edin - Nəticə sayı çox sürətlə artır
  • 🧪 Test məqsədli istifadə məqsədəuyğundur - Məhdud sayda məlumatla işləyərkən
  • ⚡ WHERE şərti ilə məhdudlaşdırın - Lazım olmayan kombinasiyaları filtirləyin
  • 🔢 Nəticə sayını əvvəlcədən hesablayın - Performans probleminin qarşısını almaq üçün

SELF JOIN üçün:

  • 🏷️ Alias (ləqəb) mütləqdir - Cədvəli iki fərqli adla çağırmaq lazımdır
  • 🔄 Dövr yarada bilər - Recursive münasibətlərdə diqqətli olun
  • 🚀 İndeksləmə performansı yaxşılaşdırır - JOIN şərtində olan sütunları indeksləyin
  • 🎯 Düzgün JOIN növünü seçin - INNER, LEFT, RIGHT və ya FULL JOIN

10. Praktik Məsləhətlər

Ümumi məsləhətlər:

  1. CROSS JOIN-dən əvvəl cədvəl ölçülərini yoxlayın - EXPLAIN və ya COUNT() istifadə edin
  2. SELF JOIN-də indeksləri optimallaşdırın - JOIN şərtindəki sütunları indeksləyin
  3. Hər iki JOIN üçün EXPLAIN istifadə edin - Sorğu planını analiz edin
  4. Limit ilə nəticə sayını məhdudlaşdırın - Xüsusilə test zamanı
  5. Subquery alternativlərini nəzərdən keçirin - Bəzən subquery daha səmərəli ola bilər
  6. CTE (Common Table Expressions) istifadə edin - Mürəkkəb SELF JOIN sorğuları üçün
💡 Performans optimallaşdırması: Hər iki JOIN növündə də, böyük cədvəllər üzərində işləyərkən, lazımsız sütunları SELECT ifadəsinə daxil etməyin və yalnız lazım olan sütunları seçin.
⚠️ Təhlükəli nümunə: 10.000 sətirli bir cədvəli özü ilə CROSS JOIN etmək 100 milyon sətir nəticə verə bilər ki, bu da serveri çökdürə bilər!
